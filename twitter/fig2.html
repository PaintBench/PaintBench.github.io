Why PaintBench

A benchmark built
for precision,
not preference.

Most image editing benchmarks ask "does this look right?" PaintBench asks "is every pixel correct?" — and measures the answer objectively.

01
⬡

Primitive Operations

20 atomic task types spanning 4 foundational categories — geometric transformation, structural manipulation, color change, and symbolic reasoning. These are the building blocks of every editing workflow, from data annotation to product visualization.

1,050 problems · 20 task types · 4 categories
02
✓

Deterministically Evaluated

Every problem has exactly one correct output image. Evaluation is pixel-level comparison via CIE ΔE₇₆ color distance — the same objectivity as running code against test cases. No model judges, no human raters, no subjectivity.

Like HumanEval for code — but for images
03
∞

Dynamically Scalable

Problems are generated procedurally from random seeds with tunable parameters: canvas size, object count, background style, color palette. Fresh unseen problems can be generated at any time, preventing dataset saturation.
