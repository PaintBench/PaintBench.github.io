Visual Editing Benchmark · NYU

PaintBench

Tasks trivial in any raster editor —
still out of reach for frontier AI.

A benchmark of 1,050 deterministic visual editing problems across 20 task types. Problems are generated procedurally so they cannot be memorized. Evaluation is pure pixel math — no model judges, no ambiguity.

The best image-editing model scores only 23.5%.

Ground Truth

Primitive Operations

20 atomic task types across 4 categories: geometric transformation, structural manipulation, color change, and symbolic reasoning — the building blocks of every editing workflow.

Deterministically Evaluated

Every problem has exactly one correct output. Evaluation is pixel-level comparison — like running code against test cases, but for images. No model-based judges and their biases.

Dynamically Scalable

Generated procedurally from random seeds with tunable parameters (canvas size, object count, background style). Fresh unseen problems can be generated at any time — preventing saturation.

Companion Benchmark

The Gap Generalizes:
TinyGrafixBench

Data visualizations are built from a grammar of geometric primitives. We tested whether the precision failures revealed by PaintBench compound in a domain with immediate practical relevance.

TinyGrafixBench applies the same procedurally generated, deterministically evaluated framework to 5 chart types — bar charts, scatter plots, line charts, heatmaps, and network graphs — with 4 editing operations per chart type (construction, transformation, removal, recoloring).

Key finding: Model rankings closely mirror PaintBench. The top two models on PaintBench remain the top two on TinyGrafixBench, and the wide gap between frontier models and the rest persists across all chart types.
