01 / How a Problem Works

One correct answer.
Evaluated at the pixel.

Every PaintBench problem gives a model an input image and a precise instruction. There is exactly one ground-truth output, generated programmatically from a random seed. Evaluation compares each output pixel to the ground truth using CIEΔE76 perceptual color distance — no human opinion, no language model judge.

Instruction
Translate the circle right by 13.35% of the image width. Place the transformed shape underneath any possible overlapping shapes. Clip any parts that may extend beyond the image boundary.

02 / The Benchmark

20 tasks.
4 categories.

Tasks span a gradient from simple color operations to precise geometric transformations requiring exact spatial reasoning. Each task is fully parameterized by a random seed — 1,920 problems total (20 tasks × 8 visual conditions × 12 problems each). No web-scraped data, no ambiguous outputs.

05 / Companion Benchmark

TinyGrafixBench

The same deterministic evaluation framework applied to data visualization editing. 600 problems across 5 chart types, each with 4 primitive operations (Construction, Transformation, Removal, Recoloring). Model rankings mirror PaintBench — validating the benchmark design as a proxy for real-world editing capability beyond synthetic geometric scenes.
