01 / How a Problem Works

One correct answer.
Evaluated at the pixel.

Every PaintBench problem gives a model an input image and a precise instruction. There is exactly one ground-truth output, generated programmatically from a random seed. Evaluation compares each output pixel to the ground truth using CIEΔE76 perceptual color distance — no human opinion, no language model judge.

Instruction
Translate the circle right by 13.35% of the image width. Place the transformed shape underneath any possible overlapping shapes. Clip any parts that may extend beyond the image boundary.

02 / The Benchmark

20 primitive tasks.
4 categories.
Infinitely scalable.

Every problem is generated from a random seed: a new seed produces a fresh scene, fresh instruction, and a unique correct output — making memorization impossible. Tasks span a gradient from pixel-formula operations to spatial transformations requiring exact geometric reasoning. No web-scraped data. No ambiguous outputs.

⟳
Dynamic Generation
Fresh problem sets on demand. Each seed produces a unique (input, instruction, answer) triple — the benchmark cannot be contaminated by training data exposure.
✓
One Correct Answer
Every output is compared pixel-by-pixel to the ground truth via CIE ΔE₇₆ color distance. No human ratings, no LLM judges, no perceptual proxies.
◈
Atomic Operations
Tasks target the building blocks of visual editing — recoloring, transformation, removal — the same primitives a raster editor handles trivially in a single command.

05 / Companion Benchmark

TinyGrafixBench

The same deterministic evaluation framework applied to data visualization editing. 600 problems across 5 chart types, each with 4 primitive operations (Construction, Transformation, Removal, Recoloring). Model rankings mirror PaintBench — validating the benchmark design as a proxy for real-world editing capability beyond synthetic geometric scenes.
